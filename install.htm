Overview

The WDK provides a platform on which to develop, compile and test drivers for Windows. To obtain a logo for your driver, use the WLK, available separately. The WDK provides driver‑specific headers, libraries, sources, tools, and documentation for developing drivers that are compatible with the following operating systems:

·         Microsoft Windows Server® 2008

·         Microsoft Windows Vista™

·         Microsoft Windows Server® 2003

·         Microsoft Windows XP

  • Microsoft Windows 2000

The WDK build environments do not support the development of drivers for the following operating systems:

·         Microsoft Windows Millennium Edition

·         Microsoft Windows 98

·         Microsoft Windows 95

·         Microsoft Windows NT 4.0

Install the Windows Driver Kit

Use KitSetup.exe to install the WDK Development Environment, WDK Documentation, the Kernel Debugger, and Device Simulation Framework. All prerequisite software, such as .NET version 2.0, and Windows Document Explorer will be installed as well.

The WDK installer, KitSetup.exe, allows you to select the components you want to install by providing checkboxes which you can check and uncheck. On first install, all checkboxes will be empty, indicating that none of the WDK components has yet been installed. By clicking on the checkboxes you can select and deselect components for installation. As you check and uncheck components, the top right window of the KitSetup dialog will reflect the pending installation tasks to be performed when you click OK. You can install the following components using KitSetup:

Full Development Environment - Use this checkbox to control installation of the WDK Build Environments, Samples, Tools and Documentation.

Build Environments/Samples/Tools - Use this checkbox to control installation of the WDK Build Environments, Samples and Tools.

Help (Documentation Collection) - Use this checkbox to control installation the documentation for the WDK.

Microsoft Windows Debuggers - Use this checkbox to control installation of the Debugging Tools for Windows.

Device Simulation Framework - Use this checkbox to control installation of the Device Simulation Framework (DSF).

You can click Undo at any time to revert back to the WDK installation state of the target machine at KitSetup invocation. The circle icons next to the check boxes indicate the installed/installation state of each component:

  • Green icons indicate that a component is installable, but not currently installed.
  • Blue icons indicate that a component is already installed.
  • Grey icons indicate that a component cannot be installed due to incompatibilities with the target machine’s hardware or software configuration.
  • Red icons indicate that a component cannot be installed due to an error in the setup information for the component. This typically means that the distribution media has been damaged.

Detailed component status is displayed in the bottom window of the KitSetup dialog by clicking on the component name in the upper right KitSetup window.

Selecting the Help button displays this help document, install.htm. Click Cancel to exit KitSetup without performing any pending setup tasks, and the state of the target machine will remain unchanged. Click OK to continue with the installation.

Installation Location Dialog

The Installation Dialog displays the default installation location for the WDK. You may type in a different location, however, the path should not have any spaces. You can cancel this dialog and return to the KitSetup dialog at any time by clicking Cancel.

After you click OK installation continues to the End-User-License-Agreement dialog.

End-User-License-Agreement (EULA) Dialog

In order to continue WDK Installation you must read and agree to the end user license agreement displayed in the EULA dialog. To agree, read the entire agreement by scrolling the text all the way to the bottom of the document, click the “I Agree” checkbox and click OK. You will not be able to click OK until you have read to the end the EULA and checked “I Agree” checkbox. You can select the Cancel button at any time to return to the main KitSetup dialog.

After you click OK installation continues to the KitSetup Progress dialog.

KitSetup Progress Dialog

The KitSetup Progress dialog shows the list of components being installed, and also displays the installation progress. The top window shows each component that has been installed as well as the component currently being installed. If there is an error with installing a component, that status is also shown. The top progress bar shows the progress of the current component being installed. The bottom progress bar shows the overall WDK installation progress.

You can click Cancel to undo the installation process, which uninstalls any components just installed, exits the Installation Progress Dialog, and returns you to the KitSetup Dialog.

The Finish button becomes active only after all selected components have been installed.

Verify the WDK Build Environments Installed Correctly

Verify the installation succeeded.

Note: The default installation path is x:\WinDDK\<BldNum>. The icons are located at Programs\Windows Driver Kits\Build Number\.

To verify that the WDK Build Environments installed successfully

  1. Click Start.
  2. Point to All Programs.
  3. Point to Windows Driver Kits.
  4. Point to <build#>.
  5. Point to Build Environments.
  6. Point to Windows Vista and Windows Server 2008.
  7. If you are running Windows Vista, right‑click the Windows Vista and Windows Server 2008 Checked x86 Build Environment menu item and click Run as administrator. Otherwise, click Windows Vista and Windows Server 2008 Checked x86 Build Environment.
  8. At the command prompt, type build -cZ.

This build command compiles and links the complete set of installed sources. The build process can take more than 30 minutes to complete, depending on the WDK components installed and the system on which the build is being performed.

The build command should complete with no errors and no warnings.

The WDK installation process copies the release notes file from the installation media to the root directory of the WDK installation.

To read the WDK Release Notes after you install the WDK

  1. Click Start.
  2. Point to All Programs.
  3. Point to Windows Driver Kits.
  4. Point to <build#>.
  5. Point to Help.
  6. Click Release Notes.

Install Debugger(s)

The Debugging Tools for Windows package features WinDbg, a powerful debugger with a graphical interface that can debug both user‑mode and kernel‑mode code. This package also includes the debuggers KD, NTSD, CDB, and many additional tools. The documentation in this package describes the use of these debuggers and includes tips for user‑mode and kernel‑mode debugging.

Debugging Tools for Windows is available in 32‑bit and 64‑bit versions. The 32‑bit version is appropriate for most users. However, if you want to debug an application on a processor that is based on 64 bits, you should select the 64‑bit version. The 64‑bit version can be installed only on 64‑bit versions of the Windows operating system.

To obtain the most current version of Debugging Tools for Windows, see Microsoft Debugging Tools.

If you cannot install the most recent version of the Debugging Tools for Windows from the Internet, you can install the debugger that is appropriate for your platform:

·         For x86‑based platforms, run Setup_x86.exe from the Debuggers folder of the installation media.

·         For IA64‑based platforms, run Setup_ia64.exe from the Debuggers folder of the installation media.

·         For AMD86‑based platforms, run Setup_amd64.exe from the Debuggers folder of the installation media.
