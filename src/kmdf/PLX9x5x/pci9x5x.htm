PLX9x5x

SUMMARY

This sample demonstrates how to write driver for a generic PCI device using Windows Driver Framework. The target hardware for this driver is PLX9656/9653RDK-LITE board. The product kit and the hardware specification are available at http://www.plxtech.com/products/tools/rdk/9xxxrdks/9656rdk-lite.htm.

 

Following table gives a typical device and driver scenario and also lists the driver framework interfaces demonstrated in this sample:

 

Scenario

Features Demonstrated

The device is a PCI device with port, memory, interrupt and DMA resources. Device can be stopped and started at run-time and also supports low power states.

 

The driver is capable of doing concurrent read and write operations to the device but it can handle only one read or write request at any time.

·         Handling PnP & Power Events

·         Registering a Device Interface

·         Hardware resource mapping: Port, Memory & Interrupt

·         DMA Interfaces

·         Serialized Default Queue for Write requests

·         Serialized custom Queue for Read requests

·         Handling Interrupt & DPC

 

This sample works on Windows® 2000 and later operating systems.

This sample driver is a minimal driver meant to demonstrate the usage of the Windows Driver Framework. It is not intended for use in a production environment.

BUILDING THE SAMPLE

To build the sample drivers, you must first set up the DDK environment on your host machine. The “Installation and Release Notes” in the Windows 2000/XP/Server 2003 DDK has a complete description on how to do this.

Change to the directory containing the device source code, such as CD src\wdf\PLX9x5x.

Run build -ceZ, or use the macro BLD. This command invokes the Microsoft make routines to build the components. If the build succeeds, you will find the driver, PLX9x5x.sys in one of the subdirectories depending on the build environment. If it fails you can find errors and warnings in the buildxxx.err and buildxxx.err respectively, where xxx is either chk or fre depending on the build environment.  Copy the KMDF coinstaller (wdfcoinstallerMMmmm.dll), sys, INF and test binary to a floppy or to a directory of a test machine.

INSTALLATION

When you plug in the device and login into the Administrator account, the system will welcome you with a ‘Found New Hardware Dialog’.

On Windows XP and Windows Server 2003:

  1. Select 'Install from a list or specific location (Advanced)”, then click Next.
  2. Select ‘Add a new hardware device’ from the list, then click Next.
  3. Select ‘Select for the best drivers in these locations' and then select, “Include this location in the search path” check box and specify the media or the directory that contains the driver package, and then click Next.
  4. You will get a Security Alert about installing unsigned driver. Ignore that and continue the installation by clicking on “Yes”.
  5. The system will complete the installation, load the driver and start the device.

On Windows 2000:

  1. Select “Display list of known devices so that I can choose a specific device”, and then click Next.
  2. Select the “Other Class” in the Hardware type, and click Next.
  3. Click on the ‘Have Disk’ and specify the media or the directory that contains the driver files, and then click Next.
  4. You will get a warning message about installing unsigned driver. Ignore that and continue the installation.

NOTE: Please do not load this driver on a checked build Windows 2000. There is a known issue with WdfInterruptAcquireLock().

TESTING

To test the driver, run the PLX.EXE test application and following the menu option.
