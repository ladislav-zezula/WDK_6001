1394 Diagnostic Tools

Summary

The drivers present in this directory are WDF rewrite of WDM samples present under src\1394 directory. The non drivers pieces of the sample described in this document can be obtained from src\1394 directory.

This document and the associated source code describe how to interact with the 1394 driver stack.  The 1394 Diagnostic Tools consist of two sample WDF Drivers (1394vdev.sys and 1394diag.sys), a usermode DLL (1394api.dll) and a user mode application that uses interfaces provided by the DLL to interact with sample driver and the 1394 bus driver stack.

The 1394vdev.sys and 1394diag.sys source codes provide examples of sample drivers interfacing with the upper edge of the 1394 stack.  In addition to handling asynchronous and isochronous data transfers, the sample source code demonstrates the correct way to handle Plug and Play and power management I/O Request Packets (IRPs), which are the basic I/O Manager structures used to communicate with the 1394 stack. See the Microsoft® Windows® DDK for background information on the IRPs discussed in this sample.

The difference between the 1394vdev.sys and 1394diag.sys drivers is how they are enumerated.  The 1394vdev.sys driver is meant to be a virtual diagnostic driver that is loaded by using the IOCTL_IEEE1394_API_REQUEST defined in the 1394 DDI section.  The 1394diag.sys driver is meant to be a physical diagnostic device driver that is loaded of some actual 1394 device plugged into the PC.  In Windows XP 1394vdev.inf is used to load both 1394vdev.sys and 1394diag.sys drivers.

The 1394api.dll source code provides an easy to use mechanism by which the various 1394 DDIs can be accessed.  This dll can be used by user mode applications to access various 1394 routines.  Also this dll provides mechanisms for implementing loopback routines on both Asynchronous and Isochronous data.

The win1394.exe source code provides examples of a user mode application leveraging the 1394 stack to provide access to all of the various interfaces.   Win1394.exe can be used as a sample application which takes advantage of 1394 or as a testing tool to verify the various 1394 interfaces.

Building the Sample

To build the sample, run the build -ceZ command in the DDK build environment.  Samples must be built individually for each component.  The sample will produce four binaries: two WDM drivers (1394vdev.sys and 1394diag.sys), one DLL (1394api.dll), and one EXE (Win1394.exe).

Both 1394VDEV and 1394DIAG drivers are built using the same source code.

Please note that this sample only works on Windows XP and beyond, as a result the driver is blocked from building in the Windows 2000 build environment.

Installing the Sample

To install and use the sample 1394 tool set provided a test device must be used.  On Windows 2000, any other Windows PC connected via 1394 will enumerated as a 1394 Test Device, but this is no longer true in Windows XP and later OSes.  Now a test device can be loaded virtually via the menu options in win1394.exe or can be loaded physically by putting the 1394 stack into Diagnostic Mode.  Once in Diagnostic Mode all devices plugged in will be enumerated as 1394 Test Devices.  Diagnostic mode can be enabled/disabled through the win1394.exe menu options.

Installing a virtual 1394 device:

1)     Copy the 1394vdev.inf and 1394vdev.sys to a floppy or to a temporary directory on the target machine.

2)     Run Win1394.EXE on the target machine. Click on 1394 Devices menu and select Add Virtual Device. The application will send an IOCTL to the 1394 bus driver and request it to enumerate a virtual device.

3)     In response to that, you will see Found New Hardware Wizard dialog.

4)     Select Install from a list or specific location (Advanced)

5)     Select Dont search. I will choose the driver to install.

6)     Click on Have Disk and point to the floppy or the directory that contains the INF file and the driver.

7)     Select 1394 Virtual Device in the following dialog and click Next.

1)     Click on Yes when you get Security Alert about installing an unsigned driver.

2)     Finally the system will process the INF file and copy and load the driver for the device enumerated by the bus.

Installing a test 1394 device:

1)     Copy the 1394diag.inf and 1394diag.sys to a floppy or to a temporary directory on the target machine.

2)     Run Win1394.EXE on the target machine. Click on 1394 Devices menu and select Enable Diagnostic mode. The application will send an IOCTL to the 1394 bus driver and enable diagnostic mode on the controller.

3)     Now plug in any 1394 test device and you will get Found New Hardware Wizard dialog with the description of the device.

4)     Select Install from a list or specific location (Advanced)

5)     Select Dont search. I will choose the driver to install.

6)     Click on Have Disk and point to the floppy or the directory that contains the INF file and the driver.

7)     Select 1394 Virtual Device in the following dialog and click Next.

8)     Click on Yes when you get Security Alert about installing an unsigned driver.

9)     Finally the system will process the INF file and copy and load the driver for the device enumerated by the bus.
