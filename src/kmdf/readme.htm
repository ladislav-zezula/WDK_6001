Following table lists framework samples and a description of a usage scenario where this sample would be suitable as a starting point for driver writers.  You can bring up the readme file to learn more about the sample by clicking on the sample name.

Sample Name

Scenario

Toaster

Samples contained in this directory are meant to illustrate basic functionality of the Driver Framework. They are purely meant for learning purposes or building blocks for real drivers. It contains:

Func: This directoy contains a simple and featured version of function driver. The simple version is the most skeletal form of the driver you can write with Framework.

Bus: Contains the bus driver sample to illustrate how to enumerate child devices.

Filter: Shows how to write a filter driver and also how to provide a control deviceobject to have a private communication with an application.

Toastmon: This one shows how to open a device in kernel-mode using IoTarget.

 Echo

Demonstrates how to serialize all the I/O going into a driver by using Serial I/O Queue. Typically useful for driver that can handle only one I/O at a time. The sample defers the completion of request to a timer DPC. It also demonstrates how to mark the request cancelable while it's pending in the driver.

This sample is functionally similar to a WDM sample that has a single queue and uses IoStartPacket/IoStartNextPacket WDM APIs.

PCIDRV

Fully functional driver sample for a PCI device that uses port, memory, interrupt resources and scatter-gather DMA.

The device is capable of using either port or memory resources to control the device. Device can be suspended when Idle and also has the ability to wake the machine from low power state.


The driver is capable of doing concurrent read and write operations to the device. The driver is written to work for Intel Etherpress 10/100 Network controllers and it's variants.

This sample also illustrates how to configure the hardware for run time power management and wait-wake.
 

PLX9x5x

Sample is for a PCI device with port, memory, interrupt resources and supports scatter-gather DMA.


The driver illustrates how to serailize read and write requests to the device using seperate queues.

The sample is written for PLx PCI9056RDK-Lite Adapter.

This is just a simple version of PCIDRV.
AMCC5933

This directory contains samples for PCI and Non PNP ISA device.

The PCI device has port and DMA resouces. The driver is primarily meant to illustrate how to do non scatter-gather (Packet based) DMA.

The ISA device supports only port resources and it has to be specified in the INF file and the driver must be installed manually using Add Hardware Wizard. The driver illustrates how to program the hardware and poll the device using WorkItems

These samples are written for AMCC 5933/5935 adapters.

FakeModem

Sample for controller-less modem (soft-modem)  drivers. Soft modem drivers are generic WDM drivers. They handle serial IOCTLs to support TAPI interface on the top edge and use I/O resources to talk to the hardware or other WDM stack.

 

NONPNP

This sample would be useful for writing driver that doesn’t interact with any hardware. Typically, such drivers are written to provide some kernel-level services to a user application. These drivers are dynamically loaded by the application when it’s run and unloaded when it exits.
(E.g: FileMon, Regmon, DevicTree are examples of tools that use this type of driver).

NDIS-WDF

The purpose is to illustrate Miniport functionality and I/O Targets of Windows Driver Framework.  This driver exposes NDIS miniport interface at its upper edge and uses Windows Driver Framework(WDF) interfaces to interact with other WDM drivers such as USB, IEEE 1394, and serial at its lower edge.

OSRUSBFX2 The OSRUSBFX2 sample demonstrates how to perform bulk and isochronous data transfers to a generic USB device using Windows Driver Framework (WDF). This sample is written for OSR USB-FX2 Learning Kit. It contains a console test application to initiate bulk & isochronouse transfers and obtain information on the device's I/O endpoints. The application also demonstrates how to use GUID-based device names and pipe names generated by the operating system using the SetupDiXXX user-mode APIs.

RAMDISK

This sample is meant to show how to create  RAM disk driver. This Ramdisk can be used like any other disk, but the contents of the disk will be lost when the machine is shutdown.
SERIAL This sample is a rewrite of inbox WDM serial driver. The purpose of this sample is to illustrate how device level locking feature can be used to serialize all the I/O events, dpc and timer callbacks into the driver so that driver doesn't have to worry about any synchronization issues. This sample also illustrates runtime power managments.
KBFILTR This samples illustrates how to write a filter driver and expose side band PNP interface to an usermode application by enumerating a RAW PDO.
NDISPROT This samples shows how to write an NDIS protocol driver using WDF. Protocol drivers use generic I/O interface on the top edge and a NDIS interface at the bottom. They convert incoming I/O requests to NDIS packets and send it to miniport drivers and do the converse on the receive side.
1394 This sample illustrates how a client driver can use framework Request and IoTarget intefaces to interact with 1394 stack.
USBSAMP This sample demonstrates how to perform bulk and isochronous data transfers to a generic USB device.
WMISAMP This sample illustrates how a client driver can use the framework to handle WMI requests.
