SOCKETECHO

SUMMARY

This sample demonstrates how to use the User-Mode Driver Framework to write a driver and demonstrates best practices. It also demonstrates the use of a default Parallel Dsipatch Dispatch I/O Queue, Using Win32 Dispatcher , handling socket handle using Win32 File I/O target

Parts of this sample code are generated from VS 2005 ATL wizard. This sample driver is a minimal driver meant to demonstrate the usage of the User-Mode Driver Framework. It is not intended for use in a production environment.

BUILDING THE SAMPLE

Start a build environment window as described under Building and Testing the Samples the UMDF Readme file.

Change to the directory containing the device source code, such as "CD c:\WINDDK\<build number>\src\umdf\socketecho\driver" (the WINDDK path and build number may vary).

Run build -ceZ, or use the macro BLD. This command invokes the Microsoft make routines to build the components. If the build succeeds, you will find the driver, socketecho.dll in the %_BuildArch% subdirectory of the %TARGETPATH% directory specified in the Sources file. If it fails you can find errors and warnings in the build error (.err) and warning (.wrn) log files respectively (buildfre_wlh_x86.err for example).

INSTALLATION

To test the sample you will need to setup your test system as described Building and Testing the Samples the UMDF Readme file.

To install the socketecho driver on Windows Vista or XP (SP2 and above):

  1. First copy the driver binary and the socketecho.inf file to a directory on your test machine (for example c:\ socketechoSample.)
  2. Copy the UMDF coinstaller WUDFUpdate_01005.dll from c:\winddk\<bld>\redist\wdf\<arch> to the same directory (for example c:\socketechoSample).
  3. Change to the directory containing the inf and binaries (for example cd /d c:\socketechoSample.)
  4. Next run devcon.exe as follows:

devcon.exe install socketecho.inf WUDF\socketecho

DevCon can be found in the tools directory of your WDK enlistment (for example: c:\winddk\<bld>\tools\devcon\i386\devcon.exe.)

To update the socketecho driver after making any changes:

  1. Increment the version number found in the INF.  While this is not strictly necessary, it will help ensure PnP selects your new driver as a better match for the device.
  2. Copy the updated driver binary and the socketecho.inf file to a directory on your test machine (for example c:\ socketechoSample.)
  3. Change to the directory containing the inf and binaries (for example cd /d c:\ socketechoSample.)
  4. Next run devcon.exe as follows:

devcon.exe update socketecho.inf WUDF\socketecho
